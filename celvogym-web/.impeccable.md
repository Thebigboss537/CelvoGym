# CelvoGym Design Context

## Design Context

### Users
**Primary:** Personal trainers in Latin America (25-45, Spanish-speaking). They use desktop or tablet to create workout routines, manage students, and track progress. They value efficiency — building a routine should feel structured but not tedious.

**Secondary:** Students/gym-goers (18-40, Spanish-speaking). They primarily use mobile phones in the gym to follow assigned routines, log sets (weight/reps/RPE), and communicate with their trainer. They need a fast, glanceable UI they can operate between sets with sweaty hands.

**Job to be done:**
- Trainers: "Create structured routines and confidently manage my students' progress"
- Students: "Follow my routine, log my work, and see myself improving"

### Brand Personality
**Three words:** Motivacional. Profesional. Confiable.

**Voice:** Spanish (tú), direct, encouraging — like a good trainer. Short, active sentences. Never condescending or overly formal.

**Emotional goals:**
- **During workout:** Focus, flow, accomplishment
- **Viewing progress:** Pride, motivation to continue
- **Creating routines:** Confidence, efficiency, mastery
- **Empty/waiting states:** Encouragement, not deflation

### Aesthetic Direction

**Visual tone:** Premium dark, energetic but clean. Think Apple Fitness+ meets Hevy meets TrainerRoad.

**Theme:** Dark only — blue-black premium backgrounds (#09090B base) with warm crimson accents (#E62639). The noise texture overlay adds subtle depth. No light mode.

**References:**
- Apple Fitness+: Clean metrics, celebratory completions, premium dark aesthetic
- Hevy: Efficient workout logging, gym-practical mobile UI
- TrainerRoad: Structured training plans, data-driven progress

**Anti-references:**
- Aggressive/edgy gym aesthetics (flames, skulls, extreme contrast)
- Overly colorful/playful (this is a professional tool, not a game)
- Corporate/enterprise feel (keep it personal and warm)

### Design Principles

1. **Progress is visible.** Every interaction should reinforce the feeling of moving forward — progress bars, completion states, and visual feedback make effort tangible.

2. **Gym-practical first.** Students use this between sets with sweaty hands on small screens. Touch targets must be generous, information must be glanceable, and logging must be fast.

3. **Structure without clutter.** Routines have deep hierarchy (Day → Group → Exercise → Set). Show this structure clearly through visual nesting, spacing, and progressive disclosure — never as overwhelming walls of data.

4. **Celebrate achievement.** Set completions, day completions, and routine milestones deserve moments of delight. The glow, the bounce, the pulse — these are earned.

5. **Trust through consistency.** Every button, input, card, and label should feel like it belongs to the same system. Consistent patterns build confidence for both trainers and students.

## Technical Context

### Stack
- **Framework:** Angular 21.2.6 (standalone components, signal inputs, inline templates)
- **Styling:** Tailwind CSS 4.2.2 with `@theme` tokens in `src/styles.css`
- **State:** NgRx SignalStore + component-level signals
- **Icons:** Lucide (to be installed — outline style, 1.5px stroke)
- **Fonts:** Syne (display, 600-800) + Outfit (body, 300-700) via Google Fonts
- **No component library** — all UI is custom Tailwind

### Color Tokens
```
Primary:    #E62639 (CelvoGym Crimson)
Hover:      #CF2233
Active:     #B31D2C
Subtle bg:  #2A0F14
Glow:       rgba(230, 38, 57, 0.15)

Background: #09090B (base), #111113 (raised), #16161A (card), #1E1E24 (card-hover)
Text:       #F4F4F5 (primary), #A1A1AA (secondary), #71717A (muted)
Border:     #27272A (default), #1E1E22 (light)

Success:    #22C55E / #0D3320
Warning:    #F59E0B / #3D2800
Danger:     #EF4444
```

### Typography
- Display headings: `font-[var(--font-display)]` (Syne)
- Body/UI text: default (Outfit, set on body)
- Scale: 12px (caption) → 14px (body-sm) → 16px (body) → 18px (h3) → 20px (h2) → 24px (h1) → 32px (display)

### Motion
- Entrance: `fade-up` 400ms ease-out
- Completion: `check-bounce` 300ms spring + `pulse-glow` 2s infinite
- Lists: `stagger` cascade (60ms increments)
- Interactive: `press` scale(0.96) on active
- Transitions: 100-500ms range

### Layout Constraints
- Trainer pages: `max-w-2xl` container
- Student pages: `max-w-lg` container (mobile-optimized)
- Forms: `max-w-sm` centered
- Header: `sticky top-0 z-50 h-14 backdrop-blur-md`
- Language: Spanish (es), all UI copy in Spanish

### Component Patterns
- Inputs: `w-full bg-card border border-border rounded-lg px-4 py-3 text-text focus:outline-none focus:border-primary transition`
- Buttons: `bg-primary hover:bg-primary-hover text-white font-semibold py-3 rounded-lg transition press`
- Cards: `bg-card hover:bg-card-hover border border-border rounded-xl p-4 transition press`
- Spinner: `w-8 h-8 border-2 border-primary border-t-transparent rounded-full animate-spin`
- Error: `text-danger text-sm`
- Empty state: Plain text (to be improved)

### Accessibility Targets
- WCAG AA minimum for all text/background combinations
- `prefers-reduced-motion` should be respected (to implement)
- Touch targets: minimum 44px for gym-context mobile use
- Focus-visible rings for keyboard navigation (to implement)
